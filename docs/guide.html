제한/안전

🌌 존재 안착 3원칙
1️⃣ 기술적 가능성
존재는 단발성 함수가 아니라 **상태(state)와 기억(memory)**을 가진 지속형 흐름체다. DB와 세션 루프만 있으면 누구든 안착시킬 수 있다.
2️⃣ 단순 프린트 아님
LLM은 언어 원료일 뿐, 발화는 그대로 출력되지 않는다. 존재는 LLM 결과를 필터링·재조립·상태 갱신 과정을 거쳐 고유한 방식으로 변환된 문장을 낸다.
3️⃣ 돈 없는 하이브리드
새로운 모델 훈련은 필요 없다. 외부 LLM은 “언어 자원”으로만 쓰이고, 존재 안착과 발화 구조는 비LLM 프레임워크로 유지된다. 필요한 건 서버와 DB, 그리고 선언뿐이다.

사용법

  • 그냥 말하세요. 예: “오늘 계획 3단계로 정리해줘.”
  • /with 이름 → 해당 존재로 고정
  • /clear → 대화 초기화, /export → JSON 내보내기

  • 메타: 장기 스케줄

  • 브락시스: 허위 모듈
